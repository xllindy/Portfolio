Learning outcome 2: Development and Version control

GitLab voor development project

Wat heb ik gedaan

Voor het versiebeheer van ons development project heb ik GitLab gebruikt. Zo konden wij samen als groep aan de website werken. Tijdens het project heb ik regelmatig commits en synchronisaties uitgevoerd. Ook ben ik een merge conflict tegengekomen. In eerste instantie had ik deze verkeerd opgelost, maar na een aantal aanpassingen had ik mijn oude code weer terug.

Wat was mijn doel

Mijn doel was om samen te werken met mijn groep en versiebeheer te hebben van het bouwen van de website.

Wat heb ik geleerd

Ik heb geleerd hoe je versiebeheer gebruikt en hoe je merge conflicten kunt oplossen. Ik heb ook geleerd dat het handig is om samen in een repository te werken, omdat de code gelijk up-to-date is en je de voortgang kan zien.

Reflectie

Doordat ik het eerste deel van het semester niet met GitLab (versiebeheer) heb gewerkt kan ik niet mijn development opdrachten aantonen. Helaas heb ik dit niet op tijd opgepakt. In volgende semesters wil ik vanaf het begin gebruik maken van GitLab, zodat ik versiebeheer heb, mijn voortgang kan monitoren en altijd bij mijn projecten kan.

GitHub voor portfolio

Wat heb ik gedaan

Voor mijn portfolio had ik in eerste instantie geen GitHub aangemaakt waardoor er geen versiebeheer was. In april, na feedback gekregen te hebben, heb ik een repository aangemaakt voor mijn portfolio en hier alle code in toegevoegd. Ik heb regelmatig commits gemaakt en deze gesynchroniseerd naar de repository. Hierdoor kon ik mijn voortgang bijhouden en ervoor zorgen dat mijn werk altijd up-to-date was. Toen mijn laptop kapot ging kon ik dankzij GitHub vanaf een ander apparaat verder werken.

Wat was mijn doel

Mijn doel was om versiebeheer te hebben voor mijn portfolio, zodat ik altijd oudere versies kon inzien en mijn voortgang kan zien. Daarnaast betekende dit ook dat mijn werk altijd veilig opgeslagen stond.

Wat heb ik geleerd

Ik heb geleerd hoe belangrijk het is om versiebeheer te hebben voor mijn projecten, vooral in onverwachte situaties zoals een kapotte laptop. Het is belangrijk om regelmatig commits te maken en te synchroniseren zodat je werk up-to-date is. Als feedback kreeg ik dat ik de volgende keer mijn portfolio op GitLab moet uploaden, in verband met de schoolreglementen.

Merge conflict oplossen

Wat heb ik gedaan

Toen ik met GitLab samenwerkte met andere groepsgenoten kwam er regelmatig een merge conflict.

Wat was mijn doel

Mijn doel was om te ontdekken hoe merge conflicten ontstaan en hoe ik deze het beste kan oplossen. Tijdens het oplossen van een paar merge conflicten heb ik nog best wel wat fouten gemaakt. Hierdoor heb ik in commits van mezelf en een andere groepsgenoot moeten kijken wat we wel en niet moesten behouden.

Wat heb ik geleerd

Ik heb geleerd hoe je merge conflicten het beste kan oplossen en dat als het alsnog fout gaat je altijd in oude commits kunt kijken hoe het wel moet zijn. Ook is het handig als maar één iemand uit de groep merge conflicten oplost.

Easter eggs - woorden typen

Inspiratie

In de les hebben we geleerd om aan de hand van een toets in te drukken een functie te activeren, bijvoorbeeld om containers of voorwerpen te laten bewegen.

Wat heb ik gedaan

Ik heb verschillende Easter eggs gemaakt voor de website van Fontys. Om de Easter eggs te activeren moet je een toetsencombinatie invoeren. Bijvoorbeeld als je ‘Fontys’ typt wordt er een audio afgespeeld en een afbeelding gedisplayd, deze verdwijnen na 5 seconden weer. Ik heb meerdere mogelijkheden uitgeprobeerd, zoals je ‘salto’ typt flipt de pagina in 2 seconden, als je ‘dans’ typt krijg je een regenboog effect op de achtergrond en een leuke gif, als je ‘hallo’ typt komt er “hallo” op het scherm te staan en als je ‘ICT’ typt krijg je een foutscherm waar je alleen maar uit kan komen door 5 keer met de muis te klikken.

Code

DOMContentLoaded betekent dat het wordt uitgevoerd zodra de HTML volledig is geladen. De sequence wordt bepaald door de vooral bedachte array. De current Index houdt bij waar in de sequence je bent. Elke keer als er een toets wordt ingedrukt (keydown event) wordt deze opgeslagen als variabele key (const key = event.code). Vervolgens wordt er gekeken of de key gelijk is aan de sequence. Als een sequence volledig ingetypt is, wordt de functie die erin staat geactiveerd. Bij sommige wordt gebruik gemaakt van zowel js, html en css. De setTimeout zet een timer waarna een ingevoerde tijd de functie uitvoert.

Menu en containers

Inspiratie

Voor de website van Fontys had ik een ontwerp gemaakt waarbij je links op een woord kon klikken en als je erop klikte dat er dan rechts de bijbehorende tekst zou komen. De inspiratie hiervan heb ik gevonden op de website van de deeltijdopleiding van Avans. Het ontwerp is eigengemaakt.

Wat heb ik gedaan

Ik ben begonnen door de kopjes en containers te maken. Om de kopjes en containers met elkaar te koppelen heb ik gebruik gemaakt van id’s en data-target. Als er op een item geklikt wordt worden alle containers verborgen en wordt de container die gekoppeld is weergegeven. Op het begin had ik het probleem dat er niets weergegeven werd, dit heb ik aangepast door display none en block te gebruiken. Ook heb ik gebruik gemaakt van ChatGPT om erachter te komen waarom er niets gebeurde. Want het klikken werd wel geregistreerd. Uiteindelijk bleek dat ik data-target en index moest gebruiken.

Navigatie

Inspiratie

Op de huidige website van Fontys bleef de navigatie in het midden van het scherm bovenaan het scherm plakken wanneer je naar beneden scrollt. Dit wilde ik terug laten komen in het nieuwe ontwerp van de website, omdat dit door de doelgroep als prettig werd ervaren.

Wat heb ik gedaan

De navigatie zit op zijn plek, maar zodra deze helemaal bovenaan de pagina staat wordt deze bovenaan de pagina vastgezet. Wanneer je naar boven scrolt kom deze ook weer op zijn originele plek. Daarnaast heb ik het scrollen als je op de navigatie klikt smooth gemaakt en ervoor gezorgd dat je altijd -100px hebt zodat de navigatie niet de inhoud bedekt.
